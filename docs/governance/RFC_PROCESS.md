# RFC Process

## When RFC is required
- New public API
- Breaking behavior change
- Scheduling/runtime semantic change

## RFC workflow
1. Draft RFC from template: `docs/governance/rfcs/RFC-0000-template.md`.
2. Open discussion issue.
3. Minimum 7-day review window.
4. Maintainer decision with rationale.
5. Implementation PR references accepted RFC.

## Naming

- Final RFC file format: `docs/governance/rfcs/RFC-<number>-<short-title>.md`
- Example: `docs/governance/rfcs/RFC-0012-write-policy-default.md`
